Unity以及相关介绍

行文思路：Unity –> MONO –> C# –> .NET
一、游戏引擎如何把游戏与显卡连接在一起：
1：控制显卡进行绘制需要用到很多图像API，有的通过CPU进行控制，有的直接针对GPU进行控制，也有的是CPU已经绘制好，直接传递数据给GPU内存，进行显示。
目前市面上主流的图形API是DirectX和OpenGL，他们提供的API最终通过CPU，传递给GUP进行运行。
也有sharder这种直接对GPU进行的编程方式，编译后直接由GPU执行。
2：利用这些API，和方式，可以封装成各种接口，屏蔽各种操作细节，提供更加丰富的创作方案和组件。事实上，游戏引擎即建立于此。最后游戏代码通过游戏引擎提供的SDK，开发出丰富的游戏和玩法。
一套比较完整的游戏引擎，一般都会给游戏开发者们提供一个稳定的底层架构与完善的可视化创作工具。
二、Unity3D
Unity3D是有Unity Technologies 公司开发的跨平台专业游戏引擎，2004年诞生于丹麦的阿姆斯特丹，2005年将总部设在了美国旧金山，并发布了Unity1.0版本。5.X之后，Unity改用年作为版本号，例如笔者所使用的Unity的版本是：Unity 2017.4.2f2。
Unity3D的主要特性：
1、综合性的编辑
2、多平台的导出功能
3、便捷的自动资源导入
4、优质强大的图形动力：Unity对DirectX 和 OpenGL拥有高度优化的图形渲染管线。使
用shadow map技术实现动态阴影
5、着色器：Unity使用ShaderLab语言编写Shaders。
6、物理系统：Unity内置对NVidia的PhysX Physics engine支持
7、脚本系统：游戏使用基于Mono的脚本（后面会详细讲解Mono以及.NET的相关内容）
8、资源服务器：使用The Unity Assert Server进行版本控制，使用PostgreSql作为后台
9、音频与视频：音效系统基于OpenAL程式库，可以播放Ogg Vorbis的压缩音效。视频
播放采用Theora编码
10、地形编辑器
11、光影
12、互联网：Unity在多人网络连接中采用Raknet,减少网络游戏的开发工作量
三、Mono
Mono是有Novell公司开发的，一个致力于开创.NET在linux上使用的开源工程，包含了C#语言编译器，CLR，和相关的类库。.NET只能在Windows下运行，Mono可以实现跨平台运行与Linux,Unix，MacOS等平台，这也是Unity3D能跨平台的根本原因。Unity 2017.x目前支持C#和JavaScript语言，在此之前，还支持Boo,目前已经放弃了该语言（原因是使用人数太少）。
Unity的脚本都会经过编译，然后变成Mono的中间运行语言，因此在运行效率上，C#与JavaScript的运行效率是完全一样的，只是语言的特性和语法不同。
四、C#
类似于Java语言，相似度达到99%，是.NET的亲身儿子，开发该语言的人，还开发的Delphi。C#使用起来并不困难，笔者曾经有一年多的时间在使用该语言进行上位机的开发，该语言非常适合于写一些基于Windows的软件，具体详细，请自行谷歌，必应了解。
五、.NET
既然是C#和MONO，就不得不提.NET 框架。.NET Framework是微软公司开发的一个采用系统虚拟机运行的编程平台，以通用语言运行库（Common Language Runtime）为基础，支持多种言，如C#，VB.NET，C++，Python等。
1、C#.NET的核心是CLR，是一个可由多种编程语言使用的“运行时”，其核心功能包括：内存管理，程序集加载，安全性，异常处理，线程同步等。所有的编程语言在面向CLR编译器的编译都生成一个托管模块，托管模块是一个标准的32位windows可执行的文件（PE32）,或者64位的（PE32+）文件。被CLR监管执行的称为托管，否则被称为非托管
2、托管模块组成包括：CLR头，IL（中间语言），元数据
3、程序集：CLR与程序集（assembly）一起工作，程序集是一个或者多个模块与资源文件的逻辑分组。程序集是重用、安全性已经版本控制的最小单元。程序集是自描述的（self-describing）
4、执行程序集代码：CLR通过JIT（just-in-time）编译中间语言成汇编，并让CPU执行
5、CTS(common type system)通用类型系统，指定了用一种编程语言与另外的语言进行类型的沟通。

lua栈的基本操作

Lua虚拟机的栈操作
1、当我们创建一个新的lua虚拟机的时候，会创建一个数据元素为TValue的栈。我们可以通过代码来进行验证。例如我们常进行lua_pushboolean(L,b)的操作，该API是将一个bool类型的数据压入栈顶，看看实现的源码，如下（lapi.c）：
LUA_API void lua_pushboolean (lua_State L, int b) {
lua_lock(L);
setbvalue(L->top, (b != 0)); / ensure that true is 1 */
api_incr_top(L);
lua_unlock(L);
}
其中函数setbvalue的源码实现如下：

#define setbvalue(obj,x) \ { TValue *i_o=(obj); i_o->value.b=(x); i_o->tt=LUA_TBOOLEAN; }
通过源码，我们不难发现两个点：
（1）、虚拟的堆栈是以TValue来存储数据的，该结构是lua内部的通用结构，可以存储任何类型的C/C++数据类型
（2）、lua_pushboolean函数将数据压入栈顶（L->top）

Lua -> C++
lua_State L = luaL_newstate();
luaL_dofile(L,”test.lua”);
lua_getglobal(L,”str”);
string str = lua_tostring(L,-1);
cout << “str = “ << str << endl;
由代码可以看见，如果test.lua里面有一个str的字符串类型的数据，将会被C++程序访问到。
C++ -> Lua
C++:
char* strTest = “hello lua, I am C++”;
int test = 1;
lua_pushnumber(L,test);
lua_setglobal(L,”test”);
lua_pushstring(L, strTest);
lua_setglobal(L, “strTest”);
luaL_dofile(L,”test.lua”);//装载脚本文件进虚拟机，词法分析，语法分析，执行指令

print(strTest)
print(test)
由代码可以看见，在lua的脚本里面，可以直接调用test和strTest进行访问和操作。
目前C++、C有专门的工具来写，即tolua++。其他宿主语言我们在用的时候，如果需要手动注册，就会用到这些api

lua中的数据类型(一)

1、Lua中的数据类型主要有以下几类：
数字，布尔类型，字符串，关联表，nil，userdata，lua函数，C函数，THREAD类型。
http://www.runoob.com/lua/lua-data-types.html
Number:表示双精度类型的实浮点数
Boolean:包含两个值：false和true
String:字符串由一对双引号或者单引号来表示
Table:lua的表示其实是一个关联数组，数组的索引可以是数字或者是字符串。在lua里，table的创建是通过“构造表达式”来完成的，最简单的构造表达式是{}，用来创建一个空表。
Nil:只有值nil属于该类，表示一个无效值（在条件表达式中相当于false）
Userdata:表示任意存储在变量中的C数据结构
Function:由C或者lua编写的函数
Thread：表示执行的独立线路，用于执行协调程序

2、Lua通用数据结构体
在lua的源码当中，创建了一个通用的数据结构来接受数据进行计算和操作，在未确定具体的数据类型时，即以此结构为参数进行操作。
该结构体定义在lobject.h中：
typedef struct lua_TValue {
TValuefields;
} TValue;
其中TValuefields通过宏来定义：#define TValuefields Value value; int tt。
其中tt表示lua的数据类型(lua.h)：

#define LUA_TNONE (-1)

#define LUA_TNIL 0

#define LUA_TBOOLEAN 1

#define LUA_TLIGHTUSERDATA 2

#define LUA_TNUMBER 3

#define LUA_TSTRING 4

#define LUA_TTABLE 5

#define LUA_TFUNCTION 6

#define LUA_TUSERDATA 7

#define LUA_TTHREAD 8
其中value通过联合体的形式进行定义，这样做的最大好处就是在表示不同数据的同时，还做到了尽可能的节省空间
typedef union {
GCObject gc;
void p;
lua_Number n;
int b;
} Value;(lobject.h)
其中void p对应着LUA_TLIGHTUSERDATA和LUA_TUSERDATA
Lua_Number对应着LUA_TNUMBER(根据宏定义，lua_number其本质是double)
Int b 对应着booleanl类型。
GCObject gc 对应着需要进行垃圾回收处理的数据类型。其中GCObject是一个联合体，将所有需要进行垃圾回收的数据类型囊括起来，其内容如下(lstate.h):
union GCObject {
GCheader gch;
union TString ts;
union Udata u;
union Closure cl;
struct Table h;
struct Proto p;
struct UpVal uv;
struct lua_State th; / thread /
};
其中LUA_TSTRING 对应着TString，LUA_TTABLE 对应着Tabel，LUA_TFUNCTION 对应着Closure，LUA_TTHREAD 对应着lua_State.
其中GCheader其本质是：GCObject *next; lu_byte tt; lu_byte marked。可以看出，其本身是一个链表，将需要垃圾回收的数据加进链表中。Lua虚拟机每隔一定的时间，会检测，遍历一遍，将没有被引用的对象进行回收。

在具体的代码中，TValue用于统一地表示数据，而一旦知道了具体的数据类型，就需要使用具体类型。因此，代码中有很多涉及TValue与具体类型之间的转化代码。例如(lobject.h)：

#define setnilvalue(obj) ((obj)->tt=LUA_TNIL)

#define setnvalue(obj,x) \
{ TValue *i_o=(obj); i_o->value.n=(x); i_o->tt=LUA_TNUMBER; }

delphi调用lua的基本思路

最近由于工作的原因，会持续关注其他宿主语言与lua脚本语言的调用关系。下面根据自己的查阅和使用，简单说明一下delphi调用lua的基本思路。在网上去查询相关的问题时，会发现很多情况，我们并不能得到一个比较满意的回答，即便下载了一个示例，有时候也会发现根本运行不了，其实delphi调用lua的基本思路非常简单清晰。（估计是delphi已经是昨日黄花吧）
对于没有试过的人来说，最先关注的往往就是程序能否运行，该怎么用。我们一般下载一个库之后，会有四个核心的文件：
工程文件（.dpr）
lua.pas;
lualib.pas;
lua5.1.dll。
其中lua.pas和lualib.pas会被示例文件进行调用，而lua.pas/lualib.pas则会调用lua5.1.dll。不需要进行额外的搭建。我们也可以自己
创建的控制台程序，将lua.pas;lualib.pas装载进工程(Project ->add to Project),工程文件会自动生成uses的代码，将lua5.1.dll放到工程
目录下即可。然后借鉴示例代码，进行编码和测试。
delphi调用lua的基本思路：通过加载luaxx.dll库，运用函数GetProcAddress获取lua的库函数。当我们获取到lua的库函数，就可以利用这
些最基础的库函数，结合delphi语言，进一步封装成可以注册delphi变量，函数，对象等类型的接口。
这种库已经深入到lua的基本原理，以后会花大量的时间和博客来说明，比较好的是，早已经有人完成了这种库，难受的是，很多库都各有缺
点并且没有怎么维护了，常见的有:
lua4delphi(https://sourceforge.net/projects/lua4delphi/files/lua4delphi)
LuaDelphi 2010（http://blog.spreendigital.de/wp-content/uploads/2009/10/LuaDelphi2010-v1.3.zip）
pLua（http://lua-users.org/wiki/LuaInFreePascal）
我这里使用的是LuaDelphi2010-v1.3，使用的环境是delphi7。

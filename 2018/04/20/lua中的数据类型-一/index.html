lua中的数据类型(一)

1、Lua中的数据类型主要有以下几类：
数字，布尔类型，字符串，关联表，nil，userdata，lua函数，C函数，THREAD类型。
http://www.runoob.com/lua/lua-data-types.html
Number:表示双精度类型的实浮点数
Boolean:包含两个值：false和true
String:字符串由一对双引号或者单引号来表示
Table:lua的表示其实是一个关联数组，数组的索引可以是数字或者是字符串。在lua里，table的创建是通过“构造表达式”来完成的，最简单的构造表达式是{}，用来创建一个空表。
Nil:只有值nil属于该类，表示一个无效值（在条件表达式中相当于false）
Userdata:表示任意存储在变量中的C数据结构
Function:由C或者lua编写的函数
Thread：表示执行的独立线路，用于执行协调程序

2、Lua通用数据结构体
在lua的源码当中，创建了一个通用的数据结构来接受数据进行计算和操作，在未确定具体的数据类型时，即以此结构为参数进行操作。
该结构体定义在lobject.h中：
typedef struct lua_TValue {
TValuefields;
} TValue;
其中TValuefields通过宏来定义：#define TValuefields Value value; int tt。
其中tt表示lua的数据类型(lua.h)：

#define LUA_TNONE (-1)

#define LUA_TNIL 0

#define LUA_TBOOLEAN 1

#define LUA_TLIGHTUSERDATA 2

#define LUA_TNUMBER 3

#define LUA_TSTRING 4

#define LUA_TTABLE 5

#define LUA_TFUNCTION 6

#define LUA_TUSERDATA 7

#define LUA_TTHREAD 8
其中value通过联合体的形式进行定义，这样做的最大好处就是在表示不同数据的同时，还做到了尽可能的节省空间
typedef union {
GCObject gc;
void p;
lua_Number n;
int b;
} Value;(lobject.h)
其中void p对应着LUA_TLIGHTUSERDATA和LUA_TUSERDATA
Lua_Number对应着LUA_TNUMBER(根据宏定义，lua_number其本质是double)
Int b 对应着booleanl类型。
GCObject gc 对应着需要进行垃圾回收处理的数据类型。其中GCObject是一个联合体，将所有需要进行垃圾回收的数据类型囊括起来，其内容如下(lstate.h):
union GCObject {
GCheader gch;
union TString ts;
union Udata u;
union Closure cl;
struct Table h;
struct Proto p;
struct UpVal uv;
struct lua_State th; / thread /
};
其中LUA_TSTRING 对应着TString，LUA_TTABLE 对应着Tabel，LUA_TFUNCTION 对应着Closure，LUA_TTHREAD 对应着lua_State.
其中GCheader其本质是：GCObject *next; lu_byte tt; lu_byte marked。可以看出，其本身是一个链表，将需要垃圾回收的数据加进链表中。Lua虚拟机每隔一定的时间，会检测，遍历一遍，将没有被引用的对象进行回收。

在具体的代码中，TValue用于统一地表示数据，而一旦知道了具体的数据类型，就需要使用具体类型。因此，代码中有很多涉及TValue与具体类型之间的转化代码。例如(lobject.h)：

#define setnilvalue(obj) ((obj)->tt=LUA_TNIL)

#define setnvalue(obj,x) \
{ TValue *i_o=(obj); i_o->value.n=(x); i_o->tt=LUA_TNUMBER; }
